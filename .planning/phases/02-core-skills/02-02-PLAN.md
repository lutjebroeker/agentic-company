SSH command template (zelfde als 02-01, volledig pad vereist):
```bash
SKILL_PROMPT=$(cat /root/.claude/commands/${skill}.md)
/usr/local/bin/claude --no-permissions-prompt -p "$SKILL_PROMPT

Context (thread history):
${threadContext}

Vraag: ${userMessage}"
```

Slack kanaal → skill mapping:
```javascript
const channelSkillMap = {
  'agent-ceo':       'ceo',
  'agent-marketing': 'marketing',
  'agent-sales':     'sales',
  'agent-ops':       'ops',
  'agent-personal':  'personal'
};
```

Slack thread reply vereist thread_ts:
- Nieuw bericht (geen thread): `thread_ts` ontbreekt of gelijk aan `ts`
- Bestaande thread: `thread_ts` aanwezig in event
- Reply altijd met `thread_ts = event.thread_ts || event.ts` zodat antwoord in thread zit

Slack node postMessage configuratie:
```json
{
  "resource": "message",
  "operation": "post",
  "channel": "={{$json.channelId}}",
  "text": "={{$json.claudeResponse}}",
  "otherOptions": {
    "thread_ts": "={{$json.threadTs}}"
  }
}
```

Slack getMessages voor thread context (Slack node, resource: message, operation: getAll):
- Channel: channelId
- Filters: `oldest` = thread_ts, `inclusive` = true
- Retourneert thread-berichten als array

<task type="auto">
  <name>Task 2: Bouw n8n workflow "02-02-slack-agent-router"</name>
  <files>n8n workflow (via n8n MCP — geen lokaal bestand)</files>
  <action>
    Gebruik de n8n MCP om de workflow aan te maken. Gebruik Jelle's Slack User ID uit Task 1 voor de whitelist.

    **Node 1 — Slack Trigger**
    - Type: n8n-nodes-base.slackTrigger
    - Events: message (channels + groups)
    - Credentials: Slack credentials (Bot Token + Signing Secret)
    - Na activatie geeft n8n een webhook URL — configureer deze in Slack App → Event Subscriptions

    **Node 2 — IF: Whitelist check (alleen Jelle)**
    - Type: n8n-nodes-base.if
    - Condition: `{{$json.body.event.user}}` equals `{JELLE_SLACK_USER_ID}` (uit Task 1)
    - FALSE path: Stop (geen actie — bericht van iemand anders, of van de bot zelf)
    - TRUE path → Node 3

    **Node 3 — IF: Skip bot's own messages**
    - Type: n8n-nodes-base.if
    - Condition: `{{$json.body.event.bot_id}}` is empty (geen bot_id = menselijk bericht)
    - FALSE path: Stop (eigen bot-berichten negeren, voorkomt lussen)
    - TRUE path → Node 4

    **Node 4 — Code: Extract + kanaal-routing**
    - Type: n8n-nodes-base.code
    - Logica:
      ```javascript
      const event = $json.body.event;
      const channelId = event.channel;
      const messageText = event.text;
      const threadTs = event.thread_ts || event.ts; // reply in thread
      const messageTs = event.ts;

      // Haal kanaal naam op uit channelId — of gebruik een Set node met handmatige mapping
      // Slack geeft channel ID (C...) — kanaal naam moet opgehaald worden via Slack API
      // Alternatief: gebruik een hardcoded channel-ID → skill mapping
      // (Update deze mapping met de werkelijke channel IDs na aanmaken van kanalen)
      const channelSkillMap = {
        // Vervang met werkelijke Channel IDs uit Slack
        // Gebruik: Slack → kanaal → Open in kanaal-info → Channel ID onderaan
        // Voorbeeld: 'C0123456789': 'ceo'
        'PLACEHOLDER_CEO_CHANNEL_ID':       'ceo',
        'PLACEHOLDER_MARKETING_CHANNEL_ID': 'marketing',
        'PLACEHOLDER_SALES_CHANNEL_ID':     'sales',
        'PLACEHOLDER_OPS_CHANNEL_ID':       'ops',
        'PLACEHOLDER_PERSONAL_CHANNEL_ID':  'personal'
      };

      const skill = channelSkillMap[channelId] || null;

      if (!skill) {
        return [{ json: { skip: true } }];
      }

      return [{ json: {
        channelId,
        threadTs,
        messageTs,
        messageText,
        skill,
        skip: false
      }}];
      ```

    **Node 5 — IF: Bekende skill?**
    - Condition: `{{$json.skip}}` is false
    - FALSE path: Stop (bericht in onbekend kanaal)
    - TRUE path → Node 6

    **Node 6 — Slack: Haal thread context op**
    - Type: n8n-nodes-base.slack
    - Resource: message, Operation: getAll (conversations.replies)
    - Channel: `{{$json.channelId}}`
    - Thread TS: `{{$json.threadTs}}`
    - Retourneert alle berichten in thread voor context

    **Node 7 — Code: Bouw SSH command met thread context**
    - Type: n8n-nodes-base.code
    - Logica:
      ```javascript
      const skill = $('Code: Extract + kanaal-routing').first().json.skill;
      const userMessage = $('Code: Extract + kanaal-routing').first().json.messageText.replace(/"/g, '\\"');
      const channelId = $('Code: Extract + kanaal-routing').first().json.channelId;
      const threadTs = $('Code: Extract + kanaal-routing').first().json.threadTs;

      // Bouw thread context (max 5 berichten, nieuwste laatste)
      const threadMessages = $input.all().map(item => item.json);
      const contextLines = threadMessages
        .slice(-5)
        .map(m => `${m.user === 'JELLE_USER_ID' ? 'Jelle' : 'Bot'}: ${m.text}`)
        .join('\n');

      const skillPath = `/root/.claude/commands/${skill}.md`;
      const command = `SKILL_PROMPT=$(cat ${skillPath})
/usr/local/bin/claude --no-permissions-prompt -p "$SKILL_PROMPT

Thread context:
${contextLines}

Vraag: ${userMessage}"`;

      return [{ json: { sshCommand: command, channelId, threadTs, skill } }];
      ```

    **Node 8 — SSH: Roep Claude aan**
    - Type: n8n-nodes-base.ssh
    - Credentials: SSH credentials (zelfde als 02-01)
    - Command: `{{$json.sshCommand}}`

    **Node 9 — Code: Formatteer Slack antwoord**
    - Type: n8n-nodes-base.code
    - Logica:
      ```javascript
      const response = $json.stdout || 'Er is iets misgegaan. Probeer opnieuw.';
      const truncated = response.substring(0, 4000); // Slack max per block
      const channelId = $('Code: Bouw SSH command').first().json.channelId;
      const threadTs = $('Code: Bouw SSH command').first().json.threadTs;
      return [{ json: { claudeResponse: truncated, channelId, threadTs } }];
      ```

    **Node 10 — Slack: Stuur thread reply**
    - Type: n8n-nodes-base.slack
    - Resource: message, Operation: post
    - Channel: `{{$json.channelId}}`
    - Text: `{{$json.claudeResponse}}`
    - Other Options → thread_ts: `{{$json.threadTs}}`

    Na aanmaken: activeer workflow via n8n MCP `activateWorkflow`. Kopieer de webhook URL die n8n genereert voor de Slack Trigger — dit is nodig voor de Event Subscriptions configuratie in Task 3.

    Update de Channel ID mapping in Node 4 met de werkelijke Slack Channel IDs. Haal deze op via n8n Slack node (conversations.list) of via Slack → kanaal → Channel ID in de kanaalinfo.
  </action>
